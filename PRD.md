# PRD: 콘솔 테트리스 게임

## 1. 개요
- **목적**: 개인용 테트리스 게임을 단계적으로 개발한다.
- **개발 방식**: 여러 단계에 걸쳐 점진적으로 기능을 확장한다. 각 단계는 명확한 목표를 가진다.
- **본 문서 범위**: 1단계 (보드 표시, 단일 블록 제어, 블록 쌓기)

## 2. 기술 스택 / 실행 환경
- **언어**: Python
- **실행 환경**: Windows 콘솔(터미널)
- **입력 처리**: 실시간 키 입력 (표준 라이브러리 `msvcrt` 권장, Enter 불필요)
- **외부 의존성**: 가급적 표준 라이브러리만 사용

## 3. 1단계 목표
화면에 보드를 표시하고, 1×1 블록을 키보드로 제어하며, 블록이 자동 낙하하고 바닥/기존 블록 위에 쌓이는 기본 동작을 구현한다.

## 4. 기능 요구사항

### 4.1 보드
- 크기: 가로 10칸 × 세로 20칸 (테트리스 표준)
- 콘솔 화면에 보드 영역과 경계를 표시한다.
- 쌓인 블록과 현재 제어 중인 블록이 화면에 함께 표시된다.

### 4.2 블록
- 형태: 1×1 단일 블록
- 생성 위치: 보드 맨 위, 가로 중간 위치
- 블록은 항상 보드 내부에서만 존재할 수 있다 (경계를 벗어날 수 없다).

### 4.3 블록 제어 (키보드)
- `←` : 블록을 왼쪽으로 한 칸 이동
- `→` : 블록을 오른쪽으로 한 칸 이동
- 좌/우 이동은 보드 경계를 넘어갈 수 없다.
- 좌/우 이동 시 이미 쌓인 블록이 있는 칸으로는 이동할 수 없다.
- 블록이 고정되면 더 이상 키보드로 제어할 수 없다.

### 4.4 자동 낙하
- 블록은 1초에 한 칸씩 자동으로 아래로 내려간다.
- 낙하 시작 위치: 보드 맨 위 중간.

### 4.5 블록 고정 및 쌓기
- 블록은 다음 두 경우에 고정된다.
  1. 보드 바닥에 도달했을 때
  2. 이미 쌓인 블록 바로 위에 도달했을 때
- 고정된 블록은 해당 위치에 그대로 쌓여 화면에 계속 표시된다.
- 고정 즉시 새 블록이 보드 맨 위 중간 위치에 생성되어 동일한 절차(자동 낙하 → 고정 → 재생성)를 반복한다.

### 4.6 게임 오버
- 새 블록을 생성할 위치(보드 맨 위 중간)에 이미 블록이 쌓여 있어 생성할 수 없는 경우, 게임 오버로 처리한다.
- 게임 오버 시 화면에 게임 오버 메시지를 표시하고 게임을 종료한다.

### 4.7 게임 종료
- `Q` 키를 누르면 게임을 종료한다.

## 5. 1단계 완료 기준 (Acceptance Criteria)
- [ ] 콘솔에 10×20 보드가 표시된다.
- [ ] 블록이 보드 맨 위 중간에서 생성된다.
- [ ] 블록이 1초에 한 칸씩 자동으로 낙하한다.
- [ ] `←`, `→` 키로 블록을 좌우로 이동할 수 있다.
- [ ] 블록이 보드 경계를 벗어나지 않는다.
- [ ] 블록이 쌓인 블록이 있는 칸으로 이동하지 않는다.
- [ ] 블록이 바닥 또는 기존 쌓인 블록 위에 도달하면 고정되어 화면에 남는다.
- [ ] 블록이 고정되면 새 블록이 중간 상단에 생성된다.
- [ ] 생성 위치가 이미 막혀 있으면 게임 오버 메시지를 표시하고 종료한다.
- [ ] `Q` 키로 게임을 종료할 수 있다.

## 6. 향후 단계 (예정 — 추후 정의)
- 2단계 이후: 추가 기능은 단계별로 정의 예정 (예: 다양한 블록 모양, 회전, 줄 제거, 점수 등)
